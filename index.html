Redis: три независимые роли

Одно хранилище используется для трёх разных задач

🔒

Distributed Lock

Атомарный CAS(0→1) через RAtomicLong.compareAndSet(). Ключ: n3health:only-once-lock:{tenant}:orders:{orderId}:clients:{clientId}

TTL 3 дня. Если false → N3healthRequestAlreadySentException. Снимается через delete() при любом финальном статусе.

⏱️

Rate Limiter (Bucket4j)

Глобальный лимит на HTTP-запросы к ЕГИСЗ — суммарно по всем подам. Реализован через RedissonBasedProxyManager с CAS-счётчиком.

При превышении → EventLogStatusLimitExceededException → @Retryable с exponential backoff (2с → 4с → 8с...).

📚

Кэш НСИ (shadow-write)

Справочники Минздрава (коды специальностей, регионов) нужны при сборке CDA XML. Хранятся в RMap<Integer, NsiSubject>.

Обновление без простоя: записать в nsi:{id}:data:{ts} → атомарный rename через RTransaction. Инициируется при получении статуса FAIL.
